#使用hexo部署一个博客服务器

##环境准备
安装node.js
安装git
如果不会, 自行百度

##安装hexo相关的工具
全局安装 hexo-cli

npm i hexo-cli -g

##创建一个博客项目
使用hexo init <目录名>

下载时间较长, 比较耗时, 需要去github下载模板代码

hexo init hexo-blog

使用npm install安装项目依赖

cd hexo-blog

npm i

当安装完毕后, 目录中就会有一系列文件, 其中 _config.yml 存放着博客的配置信息

###_config.yml 配置项如下:

##更多配置参见官网配置文档

使用hexo new <文章标题>来写博客

hexo new frontend_niubi

执行完该命令后, 会在 ./source/_posts/目录下新建一个 frontend_niubi.md 文件

之后就可以随心所欲的写笔记了!

写完笔记后, 使用 hexo g 命令结合主题(配置文件中: theme: landscape)和笔记(./srouce/_posts/目录中)生成HTML/css/js静态文件

hexo g

执行完命令后, 会在 ./public 中生成静态文件

最后只需要执行hexo server 启动服务器, 即可看到博客了!

hexo server

默认开启端口是4000, 通过 -p 可以指定端口号

hexo server -p 80

hexo与github.io爱的结晶

Github给每个用户都提供了一个轻量级的web服务器, 每个人的域名是: 用户名.github.io

非常适合做博客, 这里就可以结合hexo生成的静态文件部署一个属于自己的个人博客

在github上新建一个仓库, 仓库名必须是github用户名.github.io

使用hexo的deploy功能完成项目的自动化部署

 在部署之前, 一定要先配置好部署的参数

hexo支持五个部署方式, 我们采取git

首先在_config.yml配置文件中的 deploy: 节点下配置一下 type

如果使用的是git部署, 必须先装一个包:

npm install hexo-deployer-git --save
装完以后修改配置:

deploy:
type: git # 类型为git
repo: # 仓库地址
branch: [branch] # 分支名
message: [message] #
配置完成后, 执行hexo g 然后执行 hexo d

1.hexo d是在将生成的静态文件,

2.push到远程刚刚建好的仓库中,

  1. 而刚刚建的那个仓库正好可以通过用户名.github.io直接访问,就是一个web服务器

    hexo g
    hexo d
